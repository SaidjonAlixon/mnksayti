## 5. DESIGN CONCEPT — "THE DISPATCH DESK"

A bright, operational, control-room aesthetic. The whole site reads like a modern logistics SaaS console, not an ad.

### 5.1. Bright & operational (the opposite of the rejected look)
- **Paper-white base** with lots of clean negative space.
- **MNK royal blue** is the structural ink: headers, key panels, the map, links.
- **Red is the signal only** — active states, alerts, primary CTAs, the live "moving" dots on the map.
- Thin **hairline grids** and subtle panel borders give it the precise, engineered feel of a dashboard.

### 5.2. Signature element — the LIVE FREIGHT MAP (home page hero)
This is what the site is remembered by, and it replaces the giant poster headline:
- A stylized **map of the U.S.** with MNK's hub cities as **pulsing nodes**.
- **Animated route arcs** travel between hubs (a glowing dot runs along each arc like a truck on a lane).
- A small **floating "load tag"** pops on an arc showing e.g. `CHI → DAL · DRY VAN · ON TIME`.
- Calm, continuous motion — ambient, not flashy.

### 5.3. Secondary signature — the LIVE DISPATCH BOARD
A compact, monospaced **status board** (like an airport departures board) that updates row by row:

```
LANE              EQUIP     STATUS        ETA
CHI → DAL         DRY VAN   ● IN TRANSIT  06:42
ATL → MIA         REEFER    ● LOADING     —
LAX → PHX         FLATBED   ● DELIVERED   ✓
```

It feels alive (rows flip/refresh) and proves MNK is actually *moving freight right now*.

### 5.4. Data as hero, type as support
Instead of type dominating the screen, the **map + board + a tight KPI strip** lead. Headlines are present but **medium-sized and confident**, never wall-sized. This is the cleanest break from the rejected poster.

### 5.5. Distinct hero per page
Each of the four pages opens with its own purposeful module, so they don't feel like clones:
- **HOME** → live map + dispatch board + KPIs.
- **ABOUT** → a coverage/lanes map + company timeline.
- **DRIVERS** → a pay & lifestyle panel with an application form right in view.
- **CONTACT** → a quote console with a stylized map pin to HQ.

## 6. DESIGN SYSTEM

### 6.1. Color palette (light, MNK brand)

| Role | Color | HEX |
|---|---|---|
| Paper / base | Soft white | `#F7F9FC` |
| Surface / cards | White | `#FFFFFF` |
| Brand blue (primary) | MNK Royal Blue | `#0A4D9C` |
| Blue (bright accent) | Highway Blue | `#1E6FBF` |
| Ink (headlines/text) | Deep Navy | `#0B2447` |
| Red (signal / CTA) | MNK Red | `#E0141A` |
| Hairline / border | Cool grey | `#E3E8F0` |
| Muted text | Slate grey | `#5B6B82` |
| Success (status) | Green | `#1FA463` |

> The page is **bright**. Blue carries structure and the map; red is reserved for live/active signals and primary buttons; green/amber appear only as live status dots on the board.

### 6.2. Typography — a new, technical personality
Deliberately **not** the heavy condensed poster face from v2.

| Use | Typeface (suggested) | Feel |
|---|---|---|
| Headlines | **Space Grotesk** (or Sora) | Precise, modern, slightly technical |
| Body | **Inter** | Clean, neutral, readable |
| Data / board / labels | **Space Mono** (or JetBrains Mono) | The "console / manifest" voice |

Headlines are **medium scale** (clamp ~28–56px), tracked tight. The mono face does a lot of the personality work in labels, the board, and coordinates.

### 6.3. Layout language
- **Panel-based dashboard grid** — content sits in clean cards/panels with hairline borders, like widgets on a console.
- 8-pt spacing, max width ~1200–1280px.
- Soft, small radii (4–8px) — engineered, not playful.
- Subtle shadows for elevation on white.

## 7. SITE ARCHITECTURE (separate pages)

Four standalone pages sharing one header + footer shell. Navigation = real page links (the active page's link is marked). No single-page scroll.

```
index.html      →  HOME      (live map + dispatch board + KPIs)
about.html      →  ABOUT     (story, coverage map, timeline, credentials)
drivers.html    →  DRIVERS   (pay, benefits, requirements, application, FAQ)
contact.html    →  CONTACT   (quote console, contact info, map pin)
```

Shared shell: sticky header (logo left · HOME / ABOUT / DRIVERS / CONTACT · `GET A QUOTE`), and a consistent footer (DOT/MC, contact, nav). A short **page-transition animation** plays on navigation so moving between pages feels smooth, not like hard reloads.

## 10. MOTION & INTERACTION

The motion personality is **calm and ambient**, not aggressive:
- Map arcs and node pulses loop continuously and softly.
- Dispatch board rows refresh on a gentle timer (e.g. a row flips every few seconds).
- On-scroll: panels **fade + rise** in subtly.
- Hover: tiles lift slightly, links underline, buttons fill.
- Page change: a quick wipe/fade transition.
- Everything respects `prefers-reduced-motion` (map freezes to a clean static state, board stops cycling).

---

## 11. TECH STACK (proposed)

- **Frontend:** semantic HTML + modern CSS + vanilla JS, one page per section + shared `styles.css` / `app.js`.
- **Map:** lightweight inline SVG of the U.S. with JS-animated arcs/nodes (no heavy mapping library needed).
- **Fonts:** Google Fonts or self-hosted (Space Grotesk / Inter / Space Mono).
- **Hosting (later):** Netlify / Vercel / shared hosting.

---

## 12. RESPONSIVE / MOBILE

- **Mobile-first.** Breakpoints: `≤480` / `481–768` / `769+`.
- On mobile: header → hamburger; the freight map becomes a simpler static/animated graphic; the dispatch board scrolls horizontally or stacks; forms are single-column with large touch targets.

## 13. SEO & PERFORMANCE

- Unique `<title>` + meta description per page (HOME/ABOUT/DRIVERS/CONTACT).
- Semantic structure, Open Graph cards, sitemap.xml.
- Optimized assets, lazy loading, preloaded critical fonts. Target Lighthouse 90+.
